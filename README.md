# ربات اعتمادسنجی تلگرام — TrustBot (MVP اولیه)

## معماری

این پروژه از دو سرویس مستقل تشکیل شده که با هم از طریق REST API صحبت می‌کنن:

```
┌─────────────────┐         REST API        ┌──────────────────────┐
│   Telegram Bot   │ ───────────────────────▶ │   Django Backend     │
│   (aiogram 3)    │ ◀─────────────────────── │  (Django + DRF)      │
└─────────────────┘                          │                      │
                                              │  - پنل ادمین         │
                                              │  - دیتابیس           │
                                              │  - منطق اصلی کسب‌وکار│
                                              └──────────────────────┘
```

**چرا این معماری؟**
- پنل ادمین قدرتمند Django رایگان میاد (بدون اینکه چیزی از صفر بسازی).
- بعداً اگه خواستی یه اپ موبایل، یه وب‌اپ (Mini App تلگرام)، یا حتی یه ربات
  دیگه (مثلاً بله یا واتس‌اپ) اضافه کنی، همه می‌تونن از همین API استفاده کنن
  — مجبور نیستی منطق رو تکرار کنی.
- جداسازی باعث می‌شه تست و دیپلوی هر بخش مستقل از بخش دیگه باشه.

## ساختار پوشه‌ها

```
trustbot/
├── core/             # تنظیمات اصلی جنگو (settings, urls)
├── accounts/         # کاربران تلگرامی + درخواست‌های تأیید هویت
├── trust/            # امتیاز اعتماد + گزارش تخلف
├── ads/              # سیستم آگهی (شروع با بار و چمدان)
├── payments/         # تراکنش‌های پرداخت (gateway-agnostic)
├── api/               # لایه‌ی REST API که ربات باهاش حرف می‌زنه
└── bot/              # سرویس مستقل ربات تلگرام (aiogram)
    ├── handlers/      # هندلرهای هر بخش (شروع، تأیید هویت، گزارش، آگهی)
    ├── api_client.py  # کلاینت HTTP برای صحبت با Django
    └── main.py        # نقطه‌ی ورود ربات
```

## راه‌اندازی (Development)

```bash
# 1. نصب پکیج‌ها
pip install -r requirements.txt

# 2. تنظیم متغیرهای محیطی
cp .env.example .env
# سپس مقادیر را داخل .env با مقادیر واقعی پر کن (BOT_TOKEN از @BotFather)

# 3. آماده‌سازی دیتابیس
python manage.py migrate
python manage.py createsuperuser

# 4. اجرای بک‌اند جنگو (در یک ترمینال)
python manage.py runserver

# 5. اجرای ربات تلگرام (در یک ترمینال جدا)
python -m bot.main
```

پنل ادمین: `http://127.0.0.1:8000/admin/`

## وضعیت فعلی

✅ ساخته و تست شده:
- مدل‌های کامل: کاربر، تأیید هویت، امتیاز اعتماد، گزارش تخلف، آگهی، پرداخت
- پنل ادمین کامل با اکشن‌های تأیید/رد دسته‌جمعی
- REST API امن (با کلید مشترک بین ربات و بک‌اند)
- **صفحه‌ی وب نمایش عمومی پروفایل** در `/p/<slug>/` — یک صفحه‌ی HTML واقعی
  و قابل‌اشتراک‌گذاری، نه فقط JSON
- **اتصال کامل درگاه زرین‌پال** (`payments/zarinpal.py`) — جریان کامل
  request → redirect → verify → ثبت نتیجه
- ربات تلگرام با جریان کامل: ثبت‌نام، تأیید هویت (مدرک + سلفی)،
  گزارش تخلف (با جست‌وجوی واقعی یوزرنیم)، ثبت آگهی
- **اعلان خودکار به کاربر** وقتی ادمین تأیید/رد می‌کنه (مستقل از روشن
  بودن پروسه‌ی ربات)
- دکمه‌ی «امتیاز من» با اطلاعات واقعی از دیتابیس

⚠️ قبل از رفتن به Production باید تکمیل بشه:
1. **اتصال نتیجه‌ی پرداخت به اکشن واقعی** — مثلاً وقتی پرداخت `ad_publish`
   موفق می‌شه، آگهی مربوطه باید خودکار منتشر بشه. زیرساختش آماده‌ست
   (`payment.purpose` / `related_object_id`)، فقط منطق هر مورد رو باید
   توی `payment_callback` اضافه کنیم.
2. **محدودسازی نرخ درخواست (rate limiting)** روی API برای جلوگیری از سوءاستفاده.
3. **انتقال از SQLite به PostgreSQL** برای production (تنظیماتش در `.env.example` آماده‌ست).
4. **Webhook به‌جای polling** برای ربات وقتی روی سرور دیپلوی شد (پایدارتر و مقیاس‌پذیرتر).
5. **پشتیبان‌گیری خودکار از دیتابیس** و رمزنگاری فیلدهای حساس (مثلاً شماره مدرک شناسایی).
6. **اعلان به ادمین در گروه/کانال تلگرام** وقتی درخواست جدید (تأیید هویت یا
   گزارش) میاد — به جای اینکه فقط منتظر چک کردن دستی پنل باشه.

## افزودن قابلیت‌های جدید (طبق نیاز "قابل توسعه بودن")

برای اضافه کردن یک نوع آگهی جدید (مثلاً «هم‌سفری»)، فقط کافیه:
```python
# ads/models.py
class AdCategory(models.TextChoices):
    LUGGAGE = "luggage", "بار و چمدان"
    RIDE = "ride", "هم‌سفری"   # <- خط جدید
```
بدون نیاز به تغییر در ساختار دیتابیس یا API.

برای اضافه کردن یک ربات دیگه (مثلاً بله) به همین سیستم، کافیه یک سرویس جدید
شبیه پوشه‌ی `bot/` بسازی که به همین `api/` وصل بشه — منطق اصلی (امتیازدهی،
تأیید هویت، آگهی) رو نباید دوباره بنویسی.
